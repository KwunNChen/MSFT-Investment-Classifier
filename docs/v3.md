# V3: trying to make it actually good

* v1 and v2 proved the four MSFT only clues are empty at every horizon: 8 lucky wins in 54 walk forward fights, zero winning records
* so V3 is not "tune the models harder" (that road leads to fooling ourselves, which this pipeline exists to prevent). V3 asks ONE new question: does MARKET CONTEXT add information the stock alone lacks?
    * Basically, MSFT does not trade in a vacuum. Maybe the S&P's trend, the Nasdaq's trend, or the market's fear level (VIX) knows something MSFT's own price history does not.

## The win condition

* a model with a WINNING record against always_up across walk forward windows (majority of windows, same rows, same grader)
    * This is because a fixed accuracy target invites p hacking: the bar moves with regime and horizon. Beating the bar most eras is the honest definition of "actually good"
* if V3 fails to get there, that gets reported exactly like v1 did: as a finding

## The new discipline

* every V3 design decision (which features survive, any tuning) gets made on a train/validation split carved from the TRAINING years
* walk forward stays the final judge and gets touched ONCE per real iteration
    * Basically, the validation slice is scratch paper we may scribble on freely. The walk forward windows stay sealed until the design is frozen

## The build order

1. Context data: pull and sanity check S&P 500 (^GSPC), Nasdaq (^IXIC), and VIX (^VIX). Verify their trading calendars align with MSFT's
2. Validation slice: extend splits.py so the training years yield a train/validation split for decision making
3. Context features: S&P N day return, VIX level and change, MSFT minus S&P relative strength. Same truncation test as v1 proves no lookahead. Feature budget stays small (about 8 total)
4. Validation gated config: decide on the validation slice only whether each context feature earns its place. Freeze the winner
5. The single walk forward judgment: run the frozen config through backtest.py and horizons.py once, compare records to v1's 0 for 14
6. The dashboard: export_dashboard.py flattens all results into CSVs under results/dashboard/ for Power BI. The .pbix report gets built by hand in Power BI Desktop (guide in docs/dashboard.md)
7. Docs: the V3 story lands in findings.md and the README headline
